Что такое Transformer и как работать с библиотекой HuggingFace?

Автор: Виктория Фирсанова
Перевод туториала: Английский

Transformer — архитектура для обработки естественного языка. Впервые представлена в статье "Attention is All You Need" (Vaswani et al., 2017).

Первоначально архитектура предназначалась для машинного перевода, однако сегодня она используется для решения практически любой задачи NLP. В том числе, Transformer — в основе систем разговорного ИИ (ChatGPT и др.).

Некоторые сферы применения Transformer

Обработка естественного языка: машинный перевод, суммаризация, разговорный искусственный интеллект и всё, что вам приходит в голову можно решить с помощью Transformer. В этом туториале мы посмотрим, как классифицировать тексты с помощью этой архитектуры.

Компьютерное зрение: генерация и распознавание изображений. Между текстами и картинками много общего! Например, модель с архитектурой Transformer CLIP извлекает из текстового описания признаки для генерации картинок. Так описание "мопс в бежевой шляпе" превращается в картинку мопса в красивом головном уборе.

Генерация кода: код — это тоже текст, просто формализованный. Модели с архитектурой Transformer отлично справляются с генерацией кода, поиском ошибок и code completion, это когда среда программирования помогает дописывать конструкции кода. Так разработка становится куда более быстрым и приятным занятием для любого разработчика!

Компоненты архитектуры Transformer

Механизм внимания (Attention): архитектура Transformer считает совместную встречаемость токенов в огромных корпусах текстов. Токены — это объединения символов в "подслова" (пары символов или отдельные слова, распространенные в языке) на основе подсчета статистики совместной встречаемости символов в разных текстах. Подсчет совместной встречаемости нужен, чтобы создать матрицу внимания. Матрица внимания — это таблица, в которой отражена сила связи между словами. Сила связи показывает, как часто слова встречаются в общих контекстах, соотносятся ли они по смыслу, возможно, описывают ли они схожие понятия и т.д.

Матрицы Queries, Keys, Values: процесс подсчета совместной встречаемости в архитектуре Transformer отличается от других моделей обработки текстов! Для него мы создаем три матрицы с названиями Queries, Keys и Values. Это таблицы в которых хранятся все токены из корпуса текстов, на котором учится Transformer. По этим таблицам мы сраниванием слова на основе их совместной встречаемости в общих контекстах и оцениваем силу связи между ними. Матричные представления помогают делать расчеты параллельно, т.е. выявлять закономерности и строить матрицу внимания для множества разных текстов одновременно. Благодаря такой системе мы обрабатываем тексты быстрее и находим глубокие, неявные признаки, задающие связи между словами.

Головы внимания: есть множество способов построить матрицу внимания в зависимости от того, с какого распределения признаков мы начинаем наш поиск связей между токенами. Этот процесс можно сравнить с созданием фотографии одного и того же места с разных позиций. В зависимости от выбранной перспективы, у нас получатся разные изображения одного и того же места. Так и при построении матрицы внимания с использованием разных инициализаций, у нас получатся разные модели текстовых данных. Чтобы получить стереопредставление, мы моделируем несколько голов внимания одновременно, т.е. моделируем несколько матриц внимания одновременно с использованием разных инициализаций, разных исходных распределений весов, признаков, обозначающих силу связи между токенами.

Вот так будет выглядеть матрица внимания для двух предложений "меня зовут Джон" и "my name is John":

Числа в матрице внимания обозначают силу связи между словами. Чем ближе к 1, тем сильнее связь. Каждое слово хотя бы немного связано с другим словом в матрице -- поэтому такая модель представляет собой распределение. Веса, т.е. признаки силы связи, распределены по матрице на основе математических законов. Математические законы -- это то, что выявляет архитектура Transformer в процессе обучения!

Модель BERT

BERT — разновидность модели с архитектурной Transformer.

BERT — это кодер, т.е. модель, которая принимает на вход текст и кодирует его так, чтобы получилось векторное (числовое) представление, в котором закодированы признаки той самой совместной встречаемости слов и смысловые связи между ними. Эти представления можно использовать для решения конкретных задач, например, для классификации текстов. Для того, что превратить кодер BERT в классификатор, используют методы файн-тюнинга, тонкой настройки моделей.

Файн-тюнинг — это процесс дообучения уже обученной модели для решения целевой задачи. Например, мы можем взять модель, которую долго обучали на данных открытой предметной области (например, это вся Wikipedia) и дообучить, тонко настроить её на относительно небольшой выборке данных узкой предметной области (например, это тексты про органическую химию).

По сути, мы берем матрицы, в которых закодированы распределения связей между разными словами в языке в целом, а потом уточняем эти представления таким образом, чтобы они описывали не мир целиком, а только ту предметную область (например, органическую химию), для которой мы настраиваем модель.

3. Загружаем данные

Для файн-тюнинга мы будем использовать бенчмарк — коллекцию ресурсов (датасетов) для обучения и оценки моделей машинного обучения — The General Language Understanding Evaluation (GLUE).

Из всей коллекции данных GLUE мы будем использовать MRPC — датасет, который состоит из пар предложений. Некоторые пары предложений эквиваленты по смыслу. Некоторые — нет. Задача модели машинного обучения — научиться отличать эквивалентные пары предложений от неэквивалентных.

from datasets import load_dataset

dataset = load_dataset("glue", "mrpc")

Токенизация.

def tokenize_data(example):
  # данные из предложений 1 и 2;
  # обрезка текстов до максимальной длины,
  # которую обрабатывает модель (для BERT это 512 символов);
  # паддинг для коротких предложений;
  # pt - PyTorch
  return tokenizer(example['sentence1'], example['sentence2'], truncation=True, padding='max_length', return_tensors="pt")

tokenized_dataset = dataset.map(tokenize_data, batched=True)

5. Конфигурация модели

Загрузим BERT для классификации последовательностей.

Выведем на экран конфигурацию модели:

  • архитектура
    • BertForMaskedLM
    • BERT для моделирования языка на основе решения задачи заполнения пропусков
    • для моделирования векторных представлений, эта модель решала задачу восстановления пропущенных элементов в тексте
  • дропаут
    • способ избежать переобучения, в котором указанная доля слоев нейронной сети "выкидывается" из первоначальной модели в процессе обучения, чтобы создать несколько вариантов "дырявих" нейросетей и в качестве финального результата использовать усредненные значения весов машинного обучения
  • чекпоинт
    • если планируете обучать нейросеть долго, то промежуточные результаты можно сохранять на чекпоинтах
  • параметры нейросети
    • функция активации — это то, что преобразовывает неоднородные параметры к виду вероятностных распределений (значений от 0 до 1, которые отражают степень силы связей между выявленными нейросетью признаками)
    • размерность вектора — количество параметров, которые ищет нейронная сеть
  • параметры Transformer
    • количество голов внимания
from transformers import BertForSequenceClassification

# прописываем ту же модель из хаба, которую мы указывали в токенизаторе

6. Тонкая настройка (дообучение) BERT

Для обучения будем использовать готовые "сборки" HuggingFace Trainer.

from transformers import Trainer, TrainingArguments

training_args = TrainingArguments(
    per_device_train_batch_size=8, # батч - кусочек данных, который мы подаем за 1 шаг обучения
    per_device_eval_batch_size=8, # батч - кусочек данных, который мы подаем за 1 шаг оценки
    num_train_epochs=3, # количество эпох обучения
    logging_dir="logs", # "журнал" обучения
    output_dir="model", # позже вы сможете выгрузить отсюда вашу готовую модель
    report_to="none"  # дополнительно можно выгрузить отчет об обучении на сервис wandb, но здесь мы не будем это делать
)

trainer = Trainer(
    model=model, # в какой переменной хранится скомпилированная модель (попробуйте заменить на custom_model!)
    args=training_args, # параметры обучения мы задали выше
    train_dataset=tokenized_dataset['train'], # путь к обучающей выборке
    eval_dataset=tokenized_dataset['validation'], # путь к проверочной выборке
)

trainer.train()

7. Оценка качества обучения

Получаем предсказания для оценки моделей. Предсказания — это матрица, количество столбцов которой совпадает с количеством классов. У нас два класса — эквивалентные и неэквивалентные пары предложений.

Матрица предсказаний содержит значения для каждой пары предложений из тестовой выборки датасета MRPC, который вывел BERT после дообучения.

predictions = trainer.predict(tokenized_dataset["test"])
predictions

Эти предсказания неудобно читать, потому что модель не выбирает "эквивалентно" или "неэквивалентно". Вместо этого она присваивает класс "эквивалентно" и "неэкививалентно" каждой паре предложений и выводит число, которое отражает правдоподобие, вероятность того, что данная пара предложение является эквивалентной и неэквивалетной.

А нам нужно оставить только одно значение, то есть выбрать один класс из двух! Для этого применяется функция "аргумент максимизации". Она оставляет только одно значение с наибольшим значением из тех, что насчитала модель.

import numpy as np

preds = np.argmax(predictions.predictions, axis=-1)
preds

Выведем один пример. Здесь, из классов 0 и 1, мы отсекаем только тот, для которого модель вывела наибольшее значение.

print("Исходные предсказания модели:", predictions.predictions[0])
print("\nПосле применения аргумента максимизации:", preds[0])

Посчитаем метрики оценки для наших предсказаний: Accuracy и F-Score.

import evaluate

metric = evaluate.load("glue", "mrpc")

metric.compute(predictions=preds, references=predictions.label_ids)

Чтобы проинтерпретировать результаты, можно визуализировать значения метрик с помощью матрицы ошибок.

from sklearn.metrics import confusion_matrix, ConfusionMatrixDisplay

disp = ConfusionMatrixDisplay(confusion_matrix=confusion_matrix(predictions.label_ids, preds),
                              display_labels=['not_equivalent', 'equivalent'])
disp.plot()
